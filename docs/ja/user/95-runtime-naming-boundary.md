# ランタイム命名境界

この文書は、利用者に見せる A2O 名と、実装内部に残る互換名の境界を説明するリファレンスである。

A2O の正式名称は Agentic AI Orchestrator であり、公開プロダクト名である。A3 は内部互換名であり、実装上のコードパス、状態保存パス、Engine CLI の内部面に現れる場合がある。

読む目的は、利用者向けの案内、プロジェクトパッケージ、診断文言でどの名前を使うべきかを判断することである。通常の導入作業で A3 名を書く必要はない。A3 名を見つけた場合は、それが内部互換の説明なのか、利用者に見せるべきでない残存なのかを切り分ける。

## 公開名

利用者向けドキュメントとコマンドでは、次の名前を使う。

- `A2O`
- `a2o`
- `a2o-agent`
- `share/a2o`
- `.work/a2o/agent`
- `refs/heads/a2o/...`
- `reference-products`

## 内部互換名

次の名前は実装詳細として残ってよい。

- `A3`
- `a3`
- `bin/a3`
- `.a3`
- 旧状態を読むために必要な legacy `refs/heads/a3/...`

通常のセットアップ文書で、利用者にこれらの名前を書かせてはならない。削除済みの公開 A3 互換入力を診断に出す場合は、`migration_required=true` と A2O の置き換え先を含める。

## 命名ルール

- 新しい公開ドキュメントでは A2O 名を使う。
- 新しいプロジェクトパッケージでは A2O 名を使う。
- 新しい CLI 操作は `a2o` を優先する。
- 内部 Ruby Engine API は、公開利用者向けの面に出ない範囲で A3 名を保持してよい。
- マイグレーション案内が用意できた公開互換 alias は削除する。

## 利用者向けランタイム

利用者向けのランタイム実行は `a2o runtime run-once`、`a2o runtime loop`、または `a2o runtime resume` を使う。内部 Engine CLI の例は通常の利用者向けセットアップ文書から除外する。
