# A2O の Bounded Context と言葉

この文書は、A2O で使う語彙と Bounded Context を定義する。ドメイン、ワークスペース、証跡、実装文書はこの用語を使う。

読む目的は、同じ言葉を別の意味で使わないようにすることである。タスク、実行、フェーズ、ワークスペース、証跡は A2O の判断単位であり、UI 表示や実装都合の名前とは分けて扱う。状態遷移の詳細は [30-core-domain-model.md](30-core-domain-model.md)、ワークスペースの詳細は [40-workspace-and-repo-slot-model.md](40-workspace-and-repo-slot-model.md) を読む。

## ランタイムの流れ上の位置づけ

この文書は、カンバンタスクがスケジューラに取り込まれ、Engine がフェーズジョブを作り、a2o-agent がジョブを実行し、証跡とカンバン状態が更新されるまでの言葉を揃える。詳細文書で出てくるタスク、実行、フェーズ、ワークスペース、プロジェクトパッケージ、運用者確認は、この Bounded Context に沿って読む。

## 設計姿勢

A2O はコード構造の都合ではなく、ドメイン上の意味によって概念を命名する。

- 先に意味を定義し、その後でクラス / モジュール / ファイル名を合わせる。
- フェーズ固有の救済ブランチをドメイン語彙に入れない。
- 公開語彙は A2O 名を使う。A3 は内部互換名として必要な場合だけ残す。

## 文脈マップ

### タスク実行の文脈

所有するもの:

- タスク種別
- フェーズ
- 実行
- 終了結果
- 再実行可否

この文脈は、タスクがライフサイクルのどこにいるか、次に何が起きるかを決める。

### ワークスペースの文脈

所有するもの:

- ワークスペース種別
- リポジトリスロット
- ソース記述子
- 成果物の所有者
- 鮮度とクリーンアップ方針

この文脈は、フェーズが使うソースツリーと、作業内容の具体化方法を決める。

### プロジェクトパッケージの文脈

所有するもの:

- パッケージ識別子
- カンバンボード名
- リポジトリスロットとラベル
- エージェントの前提条件
- フェーズコマンドとスキル
- 検証コマンドと修復コマンド
- マージの既定値

この文脈は、プロダクトが所有する設定面である。

### 運用者確認の文脈

所有するもの:

- 証跡の要約
- ブロックされた実行の診断
- 要約表示
- `describe-task` の出力
- ランタイム状態と `doctor` の出力

この文脈は、何が起きたか、運用者が次に何をすべきかを説明する。

## 中核用語

### タスク

カンバンから取り込む、または親子タスクの流れの一部として作成される作業単位。

### タスク種別

- `single`: 単独タスク。
- `child`: 親タスクの範囲の一部を変更するタスク。
- `parent`: 子タスクの集約、親タスクレビュー、親タスク検証、本流へのマージを所有する統合タスク。

### フェーズ

現在処理中の実行ステップ。公開プロジェクトパッケージは次を使う。

- `implementation`
- `review`
- `parent_review`
- `verification`
- `remediation`
- `merge`

### 実行

1 つのタスクフェーズを実行する 1 回の試行。実行はフェーズ、ワークスペース、ソース記述子、結果、証跡、ブロック詳細を記録する。

### 終了結果

実行の最終結果。例: success、blocked、failed verification、merge conflict、executor failure。

### リポジトリスロット

リポジトリに対する安定したプロジェクトパッケージ上の別名。例: `app`、`repo_alpha`、`repo_beta`。ランタイム動作はハードコードされたプロダクトパスではなく、リポジトリスロットを使う。

### ワークスペース種別

- `ticket_workspace`: 実装作業に使う。
- `runtime_workspace`: レビュー、検証、マージに使う。

### 証跡

一時ログに依存せず、運用者が何が起きたかを確認できる構造化された記録と成果物。

### ソース記述子

実行が使ったコードを定義するソース参照とワークスペース種別。

### 成果物の所有者

証跡スナップショットを所有するタスクまたは親タスク。

## 公開名

利用者には A2O 名を見せる。

- `A2O`
- `a2o`
- `a2o-agent`
- `.work/a2o`
- `refs/heads/a2o/...`

内部互換名は、必要な場合だけ実装詳細と診断に残してよい。
